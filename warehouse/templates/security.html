Your Security

You may sign your uploads with GPG using the "--sign" argument to "python setup.py upload".

Additionally you may avoid using the default HTTP authentication used on the site and instead upload using twine.

The SHA-256 hash provided with files on PyPI exists only to provide some download corruption protection. It is not intended to provide any sort of security regarding tampering. Please use GPG signing for that.

PEP 458 and PEP 480 propose using The Update Framework (TUF) to secure Warehouse against man-in-the-middle attacks and a compromise of Warehouse itself respectively.
